04

Manajemen Transaksi

Lihat semua transaksi platform dan pulihkan transaksi yang terhapus.

Cara Restore Transaksi Terhapus

1

Buka halaman Transaksi

Menu Admin → Transaksi

2

Pilih filter "Terhapus (Soft Delete)"

Di dropdown filter, pilih 🗑️ Terhapus (Soft Delete) lalu klik Cari

3

Klik tombol Pulihkan

Klik tombol ↑ Pulihkan di baris transaksi yang ingin dikembalikan. Transaksi akan aktif kembali dan muncul di dashboard user.

ℹ️
Soft delete berarti data tidak benar-benar dihapus dari database — hanya ditandai sebagai terhapus. Admin dapat memulihkan kapanpun selama data belum dibersihkan secara manual.

Pencarian Transaksi

Gunakan kolom pencarian untuk cari berdasarkan kode TXN, deskripsi, nama user, atau nomor WA. Filter aktif dan terhapus bisa dikombinasikan dengan pencarian.

06

Broadcast WhatsApp

Kirim pesan ke semua user atau nomor tertentu melalui bot WA.

⏱️
Broadcast dikirim satu per satu dengan jeda 0.3 detik antar user untuk menghindari spam. Untuk 100 user, proses memakan waktu sekitar 30 detik. Halaman tidak boleh ditutup selama proses berlangsung.
TargetCara
Semua user aktifPilih "Semua User Aktif" di dropdown — jumlah user ditampilkan
Nomor tertentuPilih "Nomor WA Tertentu" → isi nomor satu per baris (format 628xxx)

Format Pesan WA

Gunakan format WhatsApp standar:

FormatHasil
*teks*Teks tebal
_teks_Teks miring
~teks~Teks dicoret
Baris baruEnter di textarea — dipertahankan di WA
07

App Settings

Kelola konfigurasi aplikasi yang bisa diubah tanpa menyentuh file kode.

ℹ️
Perubahan App Settings langsung berlaku tanpa perlu restart server. Tersedia di menu Admin → App Settings.
SettingFungsiCatatan
fonnte_token Token API Fonnte untuk kirim pesan WA keluar Field tersembunyi (secret). Kosongkan untuk tidak mengubah.
webhook_secret Token rahasia validasi request masuk dari Fonnte Harus sama dengan token di URL webhook Fonnte. Field tersembunyi.
bot_wa_number Nomor WhatsApp bot yang ditampilkan ke user Format 628xxx. Ditampilkan di halaman Pengaturan Akun user.
bot_name Nama bot yang ditampilkan di beberapa pesan Default: KitaCatat
admin_wa_number Nomor WA admin untuk menerima notifikasi sistem dan error Format 628xxx. Digunakan untuk notifikasi keamanan dan error tracking.

Webhook URL

URL webhook lengkap (dengan token) ditampilkan di panel kanan halaman App Settings — sudah include ?token=.... Salin URL ini dan daftarkan di:

Fonnte Dashboard → Devices → klik device → Webhook URL
⚠️
Jika webhook_secret diubah, URL webhook di Fonnte harus diperbarui sesuai token baru. Jika tidak, semua pesan WA masuk akan ditolak.
08

Backup Manager

Kelola backup database dari admin panel — tanpa perlu akses cPanel.

AksiCaraCatatan
Backup otomatis Berjalan setiap hari pukul 02.00 via cron job File disimpan di folder backups/ di server
Backup manual Klik tombol Backup Sekarang di halaman Backup Manager Berguna sebelum melakukan perubahan besar
Download backup Klik tombol ⬇ Download di baris file backup File .sql ter-download langsung ke komputer
Hapus backup Klik tombol 🗑 Hapus → konfirmasi Tidak bisa dibatalkan
☁️
Backup otomatis juga diupload ke Google Drive setiap hari sebagai offsite backup. File di Google Drive disimpan selama 14 hari sebelum dihapus otomatis.
💡
Backup lokal disimpan maksimal 14 hari — file lebih lama dihapus otomatis oleh cron. Pastikan folder backups/ tidak melebihi kapasitas hosting.
09

System Check

Halaman diagnostik untuk memantau kondisi server, struktur folder, skema database, dan error log.

🔐
Halaman ini dilindungi dua lapis password — login admin biasa dan password khusus System Check yang berbeda. Jangan bagikan password System Check ke siapapun.

Yang Tersedia di System Check

InformasiKegunaan
Info ServerPHP version, memory limit, disk space, APP_URL, DB Host, DB Name, debug mode status
Error Log FilesDaftar semua file error_log di seluruh folder — klik Lihat untuk buka di halaman Logs
Struktur FolderTree view lengkap semua file di server — berguna untuk memastikan file tersedia
Skema DatabaseDaftar semua tabel dan kolom di database — berguna untuk debug query

Tombol Kunci

Setelah selesai, klik tombol 🔒 Kunci di pojok kanan banner kuning untuk mengakhiri sesi System Check. Halaman akan kembali meminta password khusus saat diakses lagi.

Support Chat

Balas pesan dari user yang menghubungi via dashboard atau via WA dengan perintah admin: ...

1

Lihat pesan masuk

Menu Admin → Support Chat. Badge merah di menu menandakan ada pesan baru yang belum dibaca.

2

Pilih percakapan user

Klik nama user di panel kiri. Pesan ditampilkan seperti tampilan chat pada umumnya.

3

Balas pesan

Ketik balasan di kolom bawah → Enter atau Ctrl+Enter untuk kirim. Balasan otomatis dikirim ke WA user sekaligus tersimpan di riwayat chat.

📱
Tombol Buka WA di pojok kanan chat membuka WhatsApp langsung ke nomor user — berguna jika perlu komunikasi di luar platform.

Reset Password Admin

Jika lupa password admin, reset manual via phpMyAdmin:

UPDATE `admins` SET `password` = '$2y$10$...' WHERE `username` = 'admin';

Hash password baru bisa dibuat dengan upload sementara file tools/hash.php berisi <?php echo password_hash('passwordbaru', PASSWORD_BCRYPT); ?>
